# Openfortivpn

使用 openfortivpn 從 ubuntu 連回 fortinate

# 一、 安裝套件

# 1. 官方 github

openfortivpn

# 2. 安裝

$ sudo apt install openfortivpn ppp

# 二、 設定並執行

# 1. 設定

預設設定檔路徑 /etc/openfortivpn/config

host = 11.11.11.11
port = 3443
username = username
password = mypass

#信任的主機，首次執行之後他會給這串
trusted-cert = 996309436074c996a07ee590cdeb7f9104819460ee2335d507f430efe053e34

# 2. 執行

設定好config後，不帶參數執行即可

$ sudo openfortivpn

每分鐘掛掉自動重啟

$ sudo openfortivpn --persistent=60000

# 3. 其他功能

$ sudo openfortivpn --help
Usage: openfortivpn [<host>:<port>] [-u <user>] [-p <pass>]
                    [--realm=<realm>] [--otp=<otp>] [--set-routes=<0|1>]
                    [--half-internet-routes=<0|1>] [--set-dns=<0|1>]
                    [--pppd-no-peerdns] [--pppd-log=<file>]
                    [--pppd-ifname=<string>] [--pppd-ipparam=<string>]
                    [--pppd-call=<name>]
                    [--pppd-plugin=<file>] [--ca-file=<file>]
                    [--user-cert=<file>] [--user-key=<file>]
                    [--trusted-cert=<digest>] [--use-syslog]
                    [--persistent=<interval>] [-c <file>] [-v|-q]
       openfortivpn --help
       openfortivpn --version

Client for PPP+SSL VPN tunnel services.
openfortivpn connects to a VPN by setting up a tunnel to the gateway at
<host>:<port>. It spawns a pppd process and operates the communication between
the gateway and this process.

Options:
  -h --help                     Show this help message and exit.
  --version                     Show version and exit.
  -c <file>, --config=<file>    Specify a custom config file (default:
                                /etc/openfortivpn/config).
  -u <user>, --username=<user>  VPN account username.
  -p <pass>, --password=<pass>  VPN account password.
  -o <otp>, --otp=<otp>         One-Time-Password.
  --realm=<realm>               Use specified authentication realm on VPN gateway
                                when tunnel is up.
  --set-routes=[01]             Set if openfortivpn should configure output roues through
                                the VPN when tunnel is up.
  --no-routes                   Do not configure routes, same as --set-routes=0.
  --half-internet-routes=[01]   Add two 0.0.0.0/1 and 128.0.0.0/1 routes with higher
                                priority instead of replacing the default route.
  --set-dns=[01]                Set if openfortivpn should add VPN name servers in
                                /etc/resolv.conf
  --no-dns                      Do not reconfigure DNS, same as --set-dns=0
  --ca-file=<file>              Use specified PEM-encoded certificate bundle
                                instead of system-wide store to verify the gateway
                                certificate.
  --user-cert=<file>            Use specified PEM-encoded certificate if the server
                                requires authentication with a certificate.
  --user-key=<file>             Use specified PEM-encoded key if the server if the
                                server requires authentication with a certificate.
  --use-syslog                  Log to syslog instead of terminal.
  --trusted-cert=<digest>       Trust a given gateway. If classical SSL
                                certificate validation fails, the gateway
                                certificate will be matched against this value.
                                <digest> is the X509 certificate's sha256 sum.
                                This option can be used multiple times to trust
                                several certificates.
  --insecure-ssl                Do not disable insecure SSL protocols/ciphers.
                                If your server requires a specific cipher, consider
                                using --cipher-list instead.
  --cipher-list=<ciphers>       Openssl ciphers to use. If default does not work
                                you can try with the cipher suggested in the output
                                of 'openssl s_client -connect <host:port>'
                                (e.g. AES256-GCM-SHA384)
  --pppd-no-peerdns             Do not ask peer ppp server for DNS server addresses
                                and do not make pppd rewrite /etc/resolv.conf
  --pppd-log=<file>             Set pppd in debug mode and save its logs into
                                <file>.
  --pppd-plugin=<file>          Use specified pppd plugin instead of configuring
                                resolver and routes directly.
  --pppd-ifname=<string>        Set the pppd interface name, if supported by pppd.
  --pppd-ipparam=<string>       Provides  an extra parameter to the ip-up, ip-pre-up
                                and ip-down scripts. See man (8) pppd
  --pppd-call=<name>            Move most pppd options from pppd cmdline to
                                /etc/ppp/peers/<name> and invoke pppd with
                                'call <name>'
  --persistent=<interval>       Run the vpn persistently in a loop and try to re-
                                connect every <interval> seconds when dropping out
  -v                            Increase verbosity. Can be used multiple times
                                to be even more verbose.
  -q                            Decrease verbosity. Can be used multiple times
                                to be even less verbose.

# 三、 疑難雜症

# 問題 1

錯誤

ERROR: Gateway certificate validation failed, and the certificate digest in not in the local whitelist. If you trust it, rerun with: ERROR: --trusted-cert b996309436074c996a07ee590cdeb7f9104819460ee2335d507f430efe053e34 ERROR: or add this line to your config file: ERROR: trusted-cert = b996309436074c996a0xxxxxxxxxxxxxx

把 trusted-cert = b996309436074c996a0xxxxxxxxxxxxxx 寫入設定檔即可

# 問題 2

錯誤

ERROR: Could not authenticate to gateway (No cookie given). #350

這是密碼錯誤/連線錯誤的意思

# 問題 3

錯誤

ERROR: /usr/sbin/pppd: No such file or directory.
